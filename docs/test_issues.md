# テスト時発見課題

実行日時: 2026-02-28

## 課題一覧

### 課題1: E2Eテストの並列実行制御

- **重要度**: 中
- **カテゴリ**: テスト基盤
- **内容**: xUnitはデフォルトでテストクラスを並列実行するため、各テストのInitializeAsyncで実行するモックデータリロード（`/api/dev/reload-mockdata`）が他のテストの実行中セッションを破壊する
- **対応**: `[Collection("E2E")]` 属性で逐次実行に制御。テスト実行時間は約2.5秒で十分高速
- **提案**: CI/CD環境でのE2Eテスト実行時もこの制約を考慮すること

### 課題2: InspectionStartResponseにResumedフラグがない

- **重要度**: 低
- **カテゴリ**: API設計
- **内容**: 検品セッション再開時、レスポンスに `resumed: true` のようなフラグがない。クライアントはsessionIdの一致でのみ再開を判断する必要がある
- **提案**: `InspectionStartResponse` に `IsResumed` フラグを追加すると、フロントエンド側で「再開しました」通知の実装が容易になる

### 課題3: 伝票照合のDenpyoTypeバリデーション

- **重要度**: 低
- **カテゴリ**: API設計
- **内容**: `POST /api/inspection/slip-verify` の `DenpyoType` フィールドに不正な値（例: "INVALID"）を送信してもエラーにならず、正常に処理される
- **提案**: DenpyoTypeの値を `NOUHINSHO` / `OKURIJOU` に限定するバリデーションを追加

### 課題4: レスポンス速度

- **重要度**: 情報
- **カテゴリ**: パフォーマンス
- **内容**: 全15テストケースが約2.5秒で完了。個別のAPIレスポンスはすべて数ms〜数十msで応答しており、パフォーマンスに問題はない
- **備考**: 本テストはローカル環境（localhost）での測定。Azure環境ではネットワークレイテンシが加算される
